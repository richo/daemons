exception_log()

This is a nice little function for debugging purposes: In case a multi-threaded ruby script exits due to an uncaught exception it may be difficult to find out where the exception came from because one cannot catch exceptions that are thrown in threads other than the main thread.

This function searches for all exceptions in memory and outputs them to STDERR (if it is connected) and to a log file in the pid-file directory.

running?()

This function implements a (probably too simle) method to detect whether the program with the pid found in the pid-file is still running. It just searches for the pid in the output of ps ax, which is probably not a good idea in some cases. Alternatives would be to use a direct access method the unix process control system.

     # File lib/daemons/application.rb, line 352
352:     def stop(no_wait = false)
353:       if not running?
354:         self.zap
355:         return
356:       end
357:       
358:       pid = @pid.pid
359:       
360:       # Catch errors when trying to kill a process that doesn't
361:       # exist. This happens when the process quits and hasn't been
362:       # restarted by the monitor yet. By catching the error, we allow the
363:       # pid file clean-up to occur.
364:       begin
365:         Process.kill(SIGNAL, pid)
366:       rescue Errno::ESRCH => e
367:         puts "#{e} #{@pid.pid}"
368:         puts "deleting pid-file."
369:       end
370:       
371:       if not no_wait
372:         if @force_kill_waittime > 0
373:           puts "#{self.group.app_name}: trying to stop process with pid #{pid}..."
374:           STDOUT.flush
375:           
376:           begin
377:             Timeout::timeout(@force_kill_waittime) {
378:               while @pid.running?
379:                 sleep(0.2)
380:               end
381:             }
382:           rescue Timeout::Error
383:             puts "#{self.group.app_name}: process with pid #{pid} won't stop, we forcefully kill it..."
384:             STDOUT.flush
385:             
386:             begin
387:               Process.kill('KILL', pid)
388:             rescue Errno::ESRCH
389:             end
390:             
391:             begin
392:               Timeout::timeout(20) {
393:                 while @pid.running?
394:                   sleep(1)
395:                 end
396:               }
397:             rescue Timeout::Error
398:               puts "#{self.group.app_name}: unable to forcefully kill process with pid #{pid}."
399:               STDOUT.flush
400:             end
401:           end
402:         end
403:         
404:         
405:       end
406:       
407:       sleep(0.1)
408:       unless @pid.running?
409:         # We try to remove the pid-files by ourselves, in case the application
410:         # didn't clean it up.
411:         begin; @pid.cleanup; rescue ::Exception; end
412:         
413:         puts "#{self.group.app_name}: process with pid #{pid} successfully stopped."
414:         STDOUT.flush
415:       end
416:       
417:     end
